# AGENTS.md

本ファイルは、AIエディタ、LLMエージェント、およびコパイロット（Claude Code, Cursor, MCP Tools等）が本プロジェクトを自律的に理解・開発するための設計指示書です。
コードの追加やリファクタリングを行う際は、必ず本ファイルに記載された**厳格な制約**を遵守してください。

---

## 🎯 プロジェクトの絶対目標

本プロジェクトは、Raspberry Pi（aarch64）上で**「常時稼働中の消費メモリを5MB以下に抑える」**ことを最優先の評価指標としています。機能の追加によりこの制約を破ることは認められません。

## 🏗️ アーキテクチャと設計制限

### 1. バックエンド (Rust / Axum)

* **メモリバジェット:** 1.5MB 〜 2.5MB (待機時)
* **非同期ランタイム:** `tokio` の軽量タスクのみ。メモリ空間を肥大化させる複雑なスレッドプールの自作は禁止。
* **データベース:** `rusqlite` による埋め込みSQLiteのみ。外部RDBMS（PostgreSQL/MySQL等）への依存コードへの書き換えは厳禁。
* **状態管理:** アプリケーション状態（DB接続など）は、`Arc<Mutex<T>>` または `Arc<RwLock<T>>` でラップし、最小限のフットプリントで同期。

### 2. フロントエンド (HTML5 / Vanilla JS)

* **依存関係の禁止:** npm パッケージ（React, Vue, Tailwind, Editor.js 等）の導入は一切禁止。すべてのフロントエンドはブラウザの標準APIのみで完結させること。
* **配信戦略:** フロントエンドコードは `src/main.rs` 内にインライン文字列（`r#""#`）としてハードコード、または `include_str!` マクロによるバイナリ埋め込みを行い、ファイルI/Oによる動的メモリ確保を排除する。

## 🛑 エージェント向け開発ルール（禁止事項）

1. **外部依存クレート（Crate）の追加制限**
   機能追加の際、安易に `Cargo.toml` に依存クレートを追加しないでください。特に、高機能なWebフレームワーク周辺ツール、大型のシリアライザ、不要なロギングライブラリ（大容量のバッファを確保するもの）はメモリを消費します。標準ライブラリまたは既存の最小構成で実装してください。

2. **DOM要素の大量生成の禁止**
   フロントエンドのJavaScriptエディタ機能を拡張する場合、仮想DOMライブラリを模したようなメモリを多く消費するスクリプトの実装は避け、`contenteditable` と正規表現による軽量な文字列置換（デバウンス制御付き）を維持してください。

3. **リリースプロファイルの変更禁止**
   `Cargo.toml` に記述されている `[profile.release]` の最適化設定（`opt-level = "z"`, `lto = true`, `panic = "abort"`）は、コンパイル後のバイナリサイズと実行時メモリを極限まで削るためのものです。これを変更しないでください。

## 🔄 今後の拡張アイディアと実装方針（もしタスクを依頼された場合）

エージェントが機能拡張を提案・実装する場合は、以下の軽量アプローチに従ってください。

* **複数ページのサポート:**
  データベースの `id` 主キーを活用し、URLパス（例: `/page/:id`）から対応するレコードの `content` をSQLiteから1件だけ引いてくるルーティングを追加する（メモリ増加は数百KB以下に抑えられます）。
* **通知システム（ntfy等）との連携:**
  ノートが更新された際、外部の `ntfy` サーバーに対して超軽量な非同期HTTPリクエスト（`reqwest` などの重いライブラリを避け、最小限のソケット通信か標準のcURL呼び出し等）を送信する。
